# ⚠️ 重要：A2 Hosting 環境變數設定

## 🚫 不要使用 .env 檔案！

在 A2 Hosting 上，應該使用 **cPanel Setup Node.js App** 的環境變數設定，而不是 `.env` 檔案。

---

## ✅ 正確的設定方式

### 在 cPanel → Setup Node.js App 中設定以下環境變數：

#### 1. DATABASE_URL
**變數名稱：**
```
DATABASE_URL
```

**變數值：**（使用您的資料庫資訊）
```
postgresql://pcbabyco_acsx7339:sjwd5407@127.0.0.1:5432/pcbabyco_truth_circle
```

**格式說明：**
```
postgresql://使用者名稱:密碼@127.0.0.1:5432/資料庫名稱
```

#### 2. JWT_SECRET
**變數名稱：**
```
JWT_SECRET
```

**變數值：**
```
truthcircle-secret-key-2024-production-a2hosting
```

#### 3. NODE_ENV
**變數名稱：**
```
NODE_ENV
```

**變數值：**
```
production
```

---

## 📋 設定步驟

1. **登入 cPanel**
2. 進入 **Setup Node.js App**
3. 找到您的應用程式，點擊 **Edit** 或鉛筆圖示
4. 滾動到 **Environment variables** 區域
5. 點擊 **Add variable**，加入上面三個環境變數
6. 點擊 **Save**
7. 點擊 **Restart** 重啟應用程式
8. 等待 10-20 秒後重新整理網頁

## ❓ 為什麼不用 .env 檔案？

1. A2 Hosting 的 Passenger (Node.js App) 不會自動讀取 `.env` 檔案
2. 使用 Setup Node.js App 的環境變數更安全且更容易管理
3. 重啟時會自動載入新的環境變數

**請在 Setup Node.js App 中設定環境變數，不要創建 .env 檔案！**
