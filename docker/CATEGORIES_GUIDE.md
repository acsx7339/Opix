## 🎯 分類設計原則

### ✅ 好的分類特性

1. **具體且聚焦** - 例如「科學迷思破解」比「科學」更明確
2. **引發好奇心** - 例如「歷史真相」暗示有未知的事實
3. **貼近生活** - 例如「科技與3C」讓人聯想到手機、電腦等日常用品
4. **暗示爭議性** - 例如「政治與社會」暗示會有不同觀點

### 🔥 高流量分類排行

根據一般討論平台的經驗，預期流量排序：

1. 🥇 **健康與醫療** - 人人關心，迷思多
2. 🥈 **科技與3C** - 每個人都有手機/電腦
3. 🥉 **科學迷思破解** - 滿足好奇心，易分享
4. 4️⃣ **經濟與理財** - 關係到錢，參與度高
5. 5️⃣ **環境與永續** - 現代人關心議題
6. 6️⃣ **政治與社會** - 爭議大但可能兩極化
7. 7️⃣ **歷史真相** - 較為小眾但忠實用戶多

## 🛠️ 技術細節

### 資料庫儲存
分類儲存在 `topics` 資料表的 `category` 欄位：
```sql
category VARCHAR(50) NOT NULL
```

儲存的是英文值（如 `'Politics'`、`'Health'`），前端透過 `CATEGORY_NAMES` 對應顯示中文名稱。

### 前端定義位置
- **檔案**: `/client/src/types.ts`
- **Enum定義**: `export enum Category`
- **中文對應**: `export const CATEGORY_NAMES`

### 修改方式
如要修改分類名稱，只需編輯 `types.ts` 中的 `CATEGORY_NAMES`，Docker 的 HMR 會自動重載。
